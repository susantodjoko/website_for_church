# UI Option 1 — Targeted Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add share buttons + related sermons to sermon detail, improve empty states with CTAs, add a footer contact column, add an about page contact CTA, and make event locations clickable Google Maps links.

**Architecture:** All changes are additive template/CSS edits plus one new context processor file. No new models, no migrations. The context processor injects `footer_service_times` into every template so `base.html` can render the footer contact column without touching individual views.

**Tech Stack:** Django 5, Django templates, CSS custom properties, vanilla JS (`navigator.clipboard`).

```python
def sermon_detail(request, slug):
    sermon = get_object_or_404(Sermon, slug=slug)
    video_id = _extract_youtube_id(sermon.youtube_url)
    related = Sermon.objects.exclude(pk=sermon.pk)
    if sermon.series:
        related = related.filter(series=sermon.series)
    elif sermon.topic:
        related = related.filter(topic=sermon.topic)
    related = related[:3]
    return render(request, 'website/sermon_detail.html', {
        'sermon': sermon,
        'video_id': video_id,
        'related': related,
    })
```
